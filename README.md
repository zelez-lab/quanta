# Quanta

**GPU compute and rendering for Rust. One function, every GPU, formally verified.**

```rust
use quanta::*;

#[derive(quanta::Fields)]
struct VecAdd {
    a: Vec<f32>,
    b: Vec<f32>,
    result: Vec<f32>,
}

#[quanta::kernel]
fn vector_add(d: &VecAdd) {
    let i = quark_id();
    d.result[i] = d.a[i] + d.b[i];
}

fn main() -> Result<(), QuantaError> {
    let gpu = init()?;
    let mut data = VecAdd {
        a: vec![1.0; 1_000_000],
        b: vec![2.0; 1_000_000],
        result: vec![0.0; 1_000_000],
    };
    vector_add(&gpu, &mut data, 1_000_000)?.wait()?;
    assert_eq!(data.result[0], 3.0);
    Ok(())
}
```

That struct compiles to GPU binaries for five targets at build time. At runtime, the right one runs on whatever hardware is present. Upload, bind, dispatch, readback -- all generated by `#[derive(Fields)]`.

---

## What Makes Quanta Different

1. **One-line dispatch.** Define a struct, derive `Fields`, call the kernel function. Buffer allocation, binding, dispatch, and readback are generated at compile time. No boilerplate.

2. **5 GPU backends.** Metal (Apple), SPIR-V (Vulkan), WGSL (WebGPU), PTX (NVIDIA), GCN (AMD). All compiled at build time and embedded in your binary. The right ISA loads at runtime.

3. **Formally verified.** 147 theorems proven with Lean 4, Kani, Verus, Miri, and proptest. Axiom-grounded from GPU hardware semantics to user-facing API invariants.

4. **Zero dependencies on GPU crate ecosystem.** Raw Metal FFI (`objc_msgSend`), raw Vulkan FFI. No ash, no wgpu, no objc crate.

5. **Fast.** 120x GPU/CPU on compute-heavy math, 67x on Mandelbrot, `-O3 -ffast-math` on all backends.

## Quick Start

```sh
cargo add quanta
```

### System requirements

- **Rust 1.85+** (edition 2024)
- **LLVM 22.1** (dynamically linked) -- install via `brew install llvm@22` on macOS or [apt.llvm.org](https://apt.llvm.org/) on Debian/Ubuntu
- **Quanta compiler** (one-time install):
  ```sh
  cargo install quanta-compiler
  ```
  Without it, only MSL and WGSL targets compile. With it, all five ISAs are emitted.
- **GPU runtime** for your platform (Metal ships with macOS; Vulkan via [LunarG SDK](https://vulkan.lunarg.com/); CUDA driver from NVIDIA; ROCm from AMD)

## Formal Verification

Every behavioral property in the codebase is tracked and proven. No theorem is left unproven.

| Tool | Role | Scope |
|---|---|---|
| **Lean 4** | Mathematical truth | 17 files -- opcode bijections, scan correctness, cross-emitter semantic agreement |
| **Kani** | Bounded model checking | 9 files, 129 harnesses -- wire format roundtrips, opcode exhaustiveness, format tables |
| **Verus** | Code-level proofs | 82 files -- every emitter, every driver, every API module |
| **Miri** | Memory safety | 20 tests -- raw pointer paths, field lifecycle, Arc provenance |
| **Proptest** | Property-based fuzzing | 25 properties -- IR roundtrip, wire encoding, CPU executor equivalence |

147 theorems total. Zero unproven. Full registry in [`specs/THEOREMS.md`](specs/THEOREMS.md).

### Verification chain

```
Hardware axioms (A1-A5)     assumed correct: GPU executes submitted shader
        |
    Lean 4 specs            mathematical properties (the WHAT)
        |
    Verus proofs            code matches spec (the HOW)
        |
    Kani harnesses          no counterexample for bounded inputs (the WHEN)
        |
    Integration tests       real GPUs produce expected output (the WHERE)
```

## GPU Backends

```
Rust function
  -> #[quanta::kernel] proc macro
    -> Quanta IR (KernelDef)
      -> SPIR-V     (Vulkan)
      -> MSL        (Apple Metal)
      -> WGSL       (WebGPU)
      -> PTX        (NVIDIA via LLVM)
      -> GCN        (AMD via LLVM)
```

All five targets are emitted at build time by `quanta-compiler` (backed by LLVM 22.1) and embedded as `static` byte arrays. At runtime, Quanta detects the GPU vendor and loads the matching binary. No runtime shader compilation. No surprise failures on a different platform.

| Backend | Cargo feature | Compute | Render |
|---|---|---|---|
| Apple Metal | `metal` (default) | Yes | Yes |
| Vulkan | `vulkan` | Yes | Yes |
| Software (CPU) | `software` | Yes | -- |

NVIDIA and AMD targets do not need a separate backend feature -- they consume the PTX/GCN ISA emitted by the compiler, dispatched through the platform's CUDA or ROCm runtime.

## Rendering

The same macro system extends to vertex and fragment shaders.

```rust
#[repr(C)]
#[derive(Copy, Clone, quanta::Vertex)]
struct MyVertex {
    pos: [f32; 3],
    color: [f32; 4],
}

#[quanta::vertex]
fn vs(v: &MyVertex) -> @position [f32; 4] {
    [v.pos[0], v.pos[1], v.pos[2], 1.0]
}

#[quanta::fragment]
fn fs(color: [f32; 4]) -> [f32; 4] {
    color
}
```

Draw commands use a chainable builder:

```rust
let target = gpu.render_target(1920, 1080, Format::BGRA8)?;

gpu.render(&target)?
    .clear(Color::BLACK)
    .pipeline(&pipeline)
    .vertices(0, &verts)
    .draw(3)
    .pulse()?.wait()?;
```

Vertex layout, attribute offsets, and stride are generated by `#[derive(Vertex)]`. Fragment outputs, depth testing, instanced draw, index buffers, and texture sampling are all supported.

## Compute Features

- Shared memory (`shared!` macro)
- Atomics (add, min, max, CAS, exchange)
- Subgroup/warp intrinsics (shuffle, ballot, reduce, scan, any/all)
- Push constants and uniform buffers (`#[derive(Uniforms)]`)
- Compute textures (read/write)
- Batched dispatch (multiple kernels, one submission)
- Async completion (Metal `addCompletedHandler`, Vulkan `VkFence`)
- Const generics
- f16 and cooperative matrix (MMA)
- 30+ math intrinsics (sin, cos, sqrt, exp, log, pow, clamp, ...)

## Naming

Quanta uses subatomic physics terminology instead of GPU jargon:

| Quanta | GPU hardware | Description |
|---|---|---|
| Quark | Thread / lane | Smallest unit of execution |
| Proton | Core | Executes quarks |
| Nucleus | Compute unit | Group of protons |
| Field | Buffer | Data that quarks operate on |
| Wave | Compute dispatch | A bound kernel ready to run |
| Pulse | Fence / sync | GPU completion signal |
